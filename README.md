# LLM Evaluation Pipeline

## Overview

This project evaluates the quality of answers generated by a Retrieval-Augmented Generation (RAG) system using an LLM-as-a-Judge approach.

A benchmark dataset containing questions and reference answers is used to test the RAG system. For each question:

1. The RAG pipeline generates an answer from a knowledge base.
2. An evaluation model compares the generated answer with the reference answer.
3. Scores are assigned for multiple evaluation criteria.
4. Results are aggregated into a summary report.

The purpose of the project is to explore answer quality evaluation for LLM applications and understand how generated responses can be measured beyond exact text matching.

---

## Objectives

* Build a simple RAG pipeline over a document knowledge base.
* Create a benchmark dataset of question-answer pairs.
* Evaluate generated responses using an LLM judge.
* Generate structured evaluation reports.
* Compare semantic answer quality instead of relying on string matching.

## Dataset

The project uses two data sources:

### Knowledge Base

A PDF document is used as the source of information for the RAG system.

Example:

* GCP learning notes
* Technical documentation
* Study material

### Benchmark Dataset

A JSON file containing:

```json
{
  "question": "...",
  "expected_answer": "..."
}
```

These reference answers act as ground truth for evaluation.

---

## RAG Pipeline

The RAG implementation is intentionally lightweight.

### Document Processing

* PDF loading using PyPDF
* Recursive text chunking
* Chunk overlap for context preservation

### Embeddings

Model:

```text
sentence-transformers/all-MiniLM-L6-v2
```

Reason:

* Lightweight
* Fast
* Suitable for local semantic search experiments

### Vector Store

FAISS is used to store document embeddings and perform similarity search.

### Answer Generation

Gemini generates answers using retrieved document chunks as context.

The model is instructed to answer only from retrieved information and avoid unsupported responses.

---

## Evaluation Pipeline

For each benchmark question:

1. Generate an answer using the RAG system.
2. Compare the answer against the reference answer.
3. Produce structured evaluation scores.

The evaluator uses Gemini as a judge and returns:

### Correctness

Measures factual agreement with the reference answer.

Score Range:

```text
1 - 10
```

### Completeness

Measures how much important information is covered.

Score Range:

```text
1 - 10
```

### Relevance

Measures how directly the answer addresses the question.

Score Range:

```text
1 - 10
```

### Hallucination Risk

Categorical assessment:

```text
Low
Medium
High
```

### Reasoning

Short explanation for the assigned scores.

## Output

For every benchmark question:

```json
{
  "question": "...",
  "generated_answer": "...",
  "correctness": 9,
  "completeness": 8,
  "relevance": 10,
  "hallucination_risk": "Low"
}
```

A final summary report includes:

* Average Correctness
* Average Completeness
* Average Relevance
* Overall Score
* Hallucination Distribution

Results are stored in:

```text
outputs/evaluation_results.json
```
